前端面试题解题

如何解决跨域问题

  • jsonp
  • CORS
    • 服务端对于CORS的支持 通过设置Access-Control-Allow-Origin来进行
  • 通过修改document.domain来跨子域
    • 将子域和主域的document.domain设为同一个主域
      • 这两个域名必须属于同一个基础域名 而且所用的协议 端口都要一致
  • window.name(兼容几乎所有浏览器)
    • 该属性有个特征: 在一个窗口(window)的生命周期内 窗口载入的所有页面都是共享一个window.name的 每个页面对window.name都有读写的权限 window.name是持久存在于 一个窗口载入过的所有页面中的
  • HTML5中新引进的window.postMessage
  • flash
  • 在服务器上设置代理页面

XSS原理及防范

  • 原理
    • 是指攻击者往Web页面里插入恶意html标签或者javascript代码 然后用户提交表单的时候 把信息传送到攻击者的服务器 或者窃取cookie中用户的私密信息
  • 防范
    • 首先代码里对用户输入的地方和变量都需要仔细检查长度和对”<”,”>”,”;”,”’”等字符做过滤；其次任何内容写到页面之前都必须加以encode，避免不小心把html tag 弄出来。这一个层面做好，至少可以堵住超过一半的XSS 攻击。
    • 避免直接在cookie中泄露用户隐私
      • 还可以使cookie和系统ip绑定来降低cookie泄露后的危险 获取ip地址可以在后端php或者其他方法获取 前端获取ip地址似乎有点问题
    • 如果网站不需要浏览器对cookie进行操作 可以在Set-cookie末尾加上HttpOnly
    • 尽量采用POST而非GET

XSS与CSRF

  • 区别
    • XSS是获取信息 不需要提前知道用户页面的代码和数据包
    • CSRF是替代用户完成指定的动作 需要知道用户页面的代码和数据包
      • 登录受信任站点A 并在本地生成cookie
      • 在不登出A的情况下 访问危险站点B

Worker和webSocket

  • worker
    • 通过worker = new Worker(url)加载一个JS文件来创建一个worder 返回worker实例
    • 通过worker.postMessage(data)方法来向worker发送数据
    • 绑定worker.onmessage方法来接受worker发送送过来的数据
    • 可以使用worker.terminate()来终止一个worker的执行
  • webSocket
    • 是web应用程序的传输协议 提供了双向 按序到达的数据流
    • 是一个HTML5协议
    • 连接是持久的 服务器的更新可以被及时推送给客户端 而不需要客户端去轮询

HTTP和HTTPS

  • http承载与tcp协议上 在http和tcp之间添加一个安全协议层ssl或tsl就成了https
  • http端口为80
  • https的端口为443

javascript垃圾回收机制

  • 标记清除(mark and sweep)

    • 当变量进入执行环境的时候 比如函数中声明的一个变量 垃圾回收器将其标记为进入环境当变量离开环境的时候 也就是函数执行结束的时候 将其标记为离开环境
    • 垃圾回收器会在运行的时候给存储在内存中的所有变量加上标记，然后去掉环境中的变量以及被环境中变量所引用的变量（闭包），在这些完成之后仍存在标记的就是要删除的变量了

  • 引用计数(reference counting)

    • 在低版本IE中经常会出现内存泄露，很多时候就是因为其采用引用计数方式进行垃圾回收。引用计数的策略是跟踪记录每个值被使用的次数，当声明了一个 变量并将一个引用类型赋值给该变量的时候这个值的引用次数就加1，如果该变量的值变成了另外一个，则这个值得引用次数减1，当这个值的引用次数变为0的时 候，说明没有变量在使用，这个值没法被访问了，因此可以将其占用的空间回收，这样垃圾回收器会在运行的时候清理掉引用次数为0的值占用的空间。
    • 在IE中虽然JavaScript对象通过标记清除的方式进行垃圾回收，但BOM与DOM对象却是通过引用计数回收垃圾的，也就是说只要涉及BOM及DOM就会出现循环引用问题。

前端工程的价值

  • 为简化用户使用提供技术支持（交互部分）
  • 为多个浏览器兼容性提供支持
  • 为提高用户浏览速度（浏览器性能）提供支持
  • 为跨平台或者其他基于webkit或其他渲染引擎的应用提供支持
  • 为展示数据提供支持（数据接口）

性能优化

  • 代码层面
    • 避免使用css表达式
    • 避免使用高级选择器
    • 避免使用通配选择器
  • 缓存利用
    • 缓存Ajax

    • 使用外部js和css文件以便缓存
    • 添加Expires头// TODO
    • 服务端配置Etag
    • 减少DNS查找// TODO
  • 请求数量
    • 合并样式和脚本
    • 使用css图片精灵// TODO
    • 首屏之外的图片资源按需加载
    • 静态资源延迟加载
  • 请求带宽
    • 压缩文件
    • 开启GZIP// TODO

代码优化

  • 用hash-table优化查找
  • 少用全局变量
  • 用innerHTML代替DOM操作 减少DOM操作次数 优化javascript性能
  • 用setTimeout来避免页面失去响应
  • 缓存查找DOM节点的结果
  • 避免全局查询
  • 避免使用with
  • 多个变量声明合并
  • 避免图片和iFrame等的空Src 空的Src会重新加载当前页面
  • 尽量避免在HTML标签中写Style属性 {但是Reactjs中提倡使用这种方法 因为React的思想是ui组件化}

移动端性能优化

  • 尽量使用css3动画
  • 使用touch事件替代click事件
  • 避免使用css3渐变阴影效果
  • 使用`transform: translateZ(0)来开启硬件加速
  • 不滥用float 在渲染时候计算量比较大
  • 不滥用web字体
  • 合理使用requestAnimationFrame动画替代setTimeout
  • CSS3 transtions CSS3 3D transforms opacity canvas webGL video 会触发GPU渲染
